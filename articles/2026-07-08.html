Google Deepmind adds background execution and MCP support to Gemini API managed agents

タイトル：Google Deepmind、Gemini APIのマネージドエージェントにバックグラウンド実行とMCPサポートを追加

Google Deepmindは、Gemini APIの「Managed Agents」において、開発者の利便性とシステムの柔軟性を高めるための4つの新機能を導入しました。これにより、エージェントの非同期処理や外部システムとの連携がより円滑になります。

主なアップデート内容は以下の通りです。

  • バックグラウンド実行のサポート：エージェントを非同期で実行できるようになり、常にHTTP接続を維持しておく必要がなくなりました。
  • MCP（Model Context Protocol）への対応：外部のデータベースやAPIに直接接続するためのMCPサーバーとの連携が可能になりました。
  • カスタム関数とサンドボックスツールの共存：標準的なサンドボックスツールに加え、開発者が独自のカスタム関数を組み合わせて利用できるようになります。
  • 状態を維持した認証情報の更新：トークンなどの資格情報をリフレッシュする際にも、サンドボックス内の現在の状態（ステート）を失わずに処理を継続できます。

これらの新機能はGemini Interactions APIを通じて提供され、JavaScript、Python、cURLのサンプルコードがドキュメント内で提供されています。

sqlite-utils 4.0, now with database schema migrations

タイトル：sqlite-utils 4.0 リリース：スキーママイグレーション機能の統合

  • メジャーアップデートによる主要な新機能: sqlite-utils 4.0では、2020年以来の大きな更新として、「データベースのスキーママイグレーション」「ネストされたトランザクション（db.atomic()メソッド）」「複合外部キーへの対応」の3つの重要な新機能が導入されました。
  • Pythonによる宣言的な移行管理: Pythonライブラリを活用することで、テーブルの作成やカラムの追加、型の変更といった一連のデータベース構造の変化を定義する「マイグレーションファイル」を作成し、順次適用できるようになります。
  • 高度な table.transform() メソッドの実装: SQLite標準のALTER TABLE文では対応が難しい複雑なスキーマ変更を可能にするため、`table.transform()`が提供されます。これは一時テーブルへのデータコピーと入れ替えを行う手法を自動で処理します。
  • マイグレーション状況の自動追跡: システムは内部的に専用の管理用テーブル（_sqlite_migrations）を使用しており、どのマイグレーションが適用済みかを正確に記録し、データベースの一貫性を保ちます。
